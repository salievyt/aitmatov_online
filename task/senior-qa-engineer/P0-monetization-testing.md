# Тестирование системы монетизации

**Приоритет:** P0 (Critical)  
**Срок:** 2-3 недели  
**Роль:** Senior QA Engineer  
**Связанные задачи:** flutter-senior-architect/P0-payment-architecture.md

## Обзор
Комплексное тестирование системы подписок и платежей для обеспечения надёжности и безопасности монетизации.

## Scope тестирования

### 1. Функциональное тестирование

#### Subscription Flow
- [ ] Регистрация нового пользователя → Free tier активен
- [ ] Активация 7-дневного trial
- [ ] Конверсия trial → paid subscription
- [ ] Отмена подписки
- [ ] Восстановление покупки (iOS/Android)
- [ ] Синхронизация статуса подписки между устройствами
- [ ] Истечение trial периода → downgrade to Free
- [ ] Истечение платной подписки → downgrade to Free

#### Payment Flow
**MegaPay:**
- [ ] Успешный платёж (месячная подписка)
- [ ] Успешный платёж (годовая подписка)
- [ ] Отмена платежа пользователем
- [ ] Ошибка платежа (недостаточно средств)
- [ ] Ошибка платежа (технический сбой)
- [ ] Возврат средств (refund)

**Stripe (если реализовано):**
- [ ] Успешный платёж картой
- [ ] 3D Secure аутентификация
- [ ] Отклонённый платёж
- [ ] Expired карта

#### Paywall Screens
- [ ] Paywall после onboarding (soft)
- [ ] Paywall после 2-го бесплатного урока (hard)
- [ ] Paywall при попытке доступа к Premium функции
- [ ] Paywall после завершения бесплатного курса (celebration)
- [ ] Кнопка "Восстановить покупку" работает
- [ ] Кнопка "Продолжить с базовым" работает

#### Access Control
- [ ] Free пользователь: доступ к 3 бесплатным курсам
- [ ] Free пользователь: доступ к первым 2 урокам каждого курса
- [ ] Free пользователь: НЕТ доступа к Premium курсам
- [ ] Free пользователь: НЕТ доступа к офлайн-режиму
- [ ] Free пользователь: НЕТ доступа к сертификатам
- [ ] Premium пользователь: доступ ко всем курсам
- [ ] Premium пользователь: доступ к офлайн-режиму
- [ ] Premium пользователь: доступ к сертификатам
- [ ] Premium пользователь: доступ к родительскому контролю

### 2. Интеграционное тестирование

#### Backend Integration
- [ ] API создания подписки возвращает корректный response
- [ ] API отмены подписки работает
- [ ] API проверки статуса подписки работает
- [ ] Webhook от платёжной системы обрабатывается
- [ ] Синхронизация подписки при логине
- [ ] Синхронизация подписки при возврате в приложение

#### Analytics Integration
- [ ] Event `subscription_started` логируется
- [ ] Event `subscription_completed` логируется с ценой
- [ ] Event `subscription_cancelled` логируется
- [ ] Event `trial_started` логируется
- [ ] Event `trial_converted` логируется
- [ ] Event `paywall_viewed` логируется с source

### 3. UI/UX тестирование

#### Paywall Design
- [ ] Все тексты читаемы и без ошибок
- [ ] Кнопки кликабельны и имеют правильные размеры
- [ ] Цены отображаются корректно (299 сом/месяц)
- [ ] Сравнение Free vs Premium понятно
- [ ] Social proof отображается (количество пользователей)
- [ ] Гарантия возврата денег упомянута
- [ ] Условия использования и политика конфиденциальности доступны

#### Subscription Settings
- [ ] Текущий план отображается корректно
- [ ] Дата следующего платежа отображается
- [ ] Кнопка "Отменить подписку" работает
- [ ] Кнопка "Изменить план" работает
- [ ] История платежей отображается

#### Loading States
- [ ] Loader показывается во время обработки платежа
- [ ] Loader показывается во время проверки статуса
- [ ] Timeout обрабатывается (>30 секунд)

#### Error Handling
- [ ] Ошибка платежа: понятное сообщение пользователю
- [ ] Ошибка сети: retry механизм
- [ ] Ошибка backend: fallback UI
- [ ] Expired подписка: уведомление пользователю

### 4. Безопасность

#### Payment Security
- [ ] Платёжные данные НЕ хранятся в приложении
- [ ] Платёжные данные НЕ логируются
- [ ] HTTPS используется для всех запросов
- [ ] Токены подписки зашифрованы (FlutterSecureStorage)
- [ ] Backend валидирует подписку при каждом запросе к Premium контенту

#### Access Control Security
- [ ] Невозможно получить Premium контент без подписки (client-side bypass)
- [ ] Невозможно подделать статус подписки
- [ ] Невозможно использовать чужую подписку

### 5. Производительность

#### Response Times
- [ ] Проверка статуса подписки: <100ms
- [ ] Загрузка paywall screen: <500ms
- [ ] Инициация платежа: <2 секунды
- [ ] Подтверждение платежа: <5 секунд

#### Offline Behavior
- [ ] Статус подписки кэшируется локально
- [ ] Приложение работает офлайн с кэшированным статусом
- [ ] Синхронизация при восстановлении сети

### 6. Совместимость

#### Платформы
- [ ] iOS 13+
- [ ] Android 8.0+

#### Устройства
- [ ] iPhone (различные модели)
- [ ] iPad
- [ ] Android смартфоны (различные производители)
- [ ] Android планшеты

#### Локализация
- [ ] Русский язык
- [ ] Кыргызский язык (если поддерживается)
- [ ] Цены в сомах (KGS)

## Test Cases

### TC-001: Успешная активация trial подписки

**Preconditions:**
- Пользователь зарегистрирован
- Пользователь на Free tier
- Trial не был активирован ранее

**Steps:**
1. Открыть paywall screen
2. Нажать "Попробовать бесплатно 7 дней"
3. Выбрать метод оплаты (MegaPay)
4. Завершить платёжный flow (в sandbox режиме)

**Expected Result:**
- Подписка активирована со статусом "trialing"
- Trial end date = сегодня + 7 дней
- Пользователь имеет доступ к Premium функциям
- Event `trial_started` залогирован
- Уведомление "Trial активирован" показано

**Priority:** P0

---

### TC-002: Конверсия trial в платную подписку

**Preconditions:**
- Пользователь на trial подписке
- Trial истекает через 1 день

**Steps:**
1. Дождаться истечения trial периода (или изменить дату на backend)
2. Открыть приложение

**Expected Result:**
- Автоматическое списание 299 сом
- Подписка переходит в статус "active"
- Event `trial_converted` залогирован
- Пользователь продолжает иметь доступ к Premium

**Priority:** P0

---

### TC-003: Отмена подписки

**Preconditions:**
- Пользователь на активной платной подписке

**Steps:**
1. Открыть Settings → Subscription
2. Нажать "Отменить подписку"
3. Подтвердить отмену

**Expected Result:**
- Подписка переходит в статус "canceled"
- Доступ к Premium сохраняется до конца оплаченного периода
- Event `subscription_cancelled` залогирован
- Уведомление "Подписка отменена" показано

**Priority:** P0

---

### TC-004: Paywall блокирует доступ к Premium курсу

**Preconditions:**
- Пользователь на Free tier
- Пользователь завершил 2 бесплатных урока курса

**Steps:**
1. Открыть Premium курс
2. Попытаться начать 3-й урок

**Expected Result:**
- Paywall screen показан
- Сообщение "Вы прошли 2 из 15 уроков"
- Кнопка "Получить Premium" доступна
- Event `paywall_viewed` залогирован с source="course"

**Priority:** P0

---

### TC-005: Восстановление покупки

**Preconditions:**
- Пользователь купил подписку на устройстве A
- Пользователь установил приложение на устройство B
- Пользователь залогинился на устройстве B

**Steps:**
1. Открыть Settings → Subscription
2. Нажать "Восстановить покупку"

**Expected Result:**
- Подписка синхронизирована с устройства A
- Статус подписки "active"
- Доступ к Premium функциям восстановлен
- Уведомление "Покупка восстановлена" показано

**Priority:** P1

---

### TC-006: Ошибка платежа (недостаточно средств)

**Preconditions:**
- Пользователь на Free tier
- Баланс MegaPay < 299 сом (sandbox режим)

**Steps:**
1. Открыть paywall screen
2. Нажать "Получить Premium"
3. Выбрать MegaPay
4. Попытаться оплатить

**Expected Result:**
- Ошибка платежа от MegaPay
- Сообщение "Недостаточно средств на счёте"
- Подписка НЕ активирована
- Пользователь остаётся на Free tier
- Event `subscription_failed` залогирован

**Priority:** P1
