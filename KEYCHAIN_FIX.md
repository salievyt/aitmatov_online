# Keychain iOS Simulator Fix

## Проблема
На iOS симуляторе Keychain недоступен (ошибка -34018), что приводило к ошибкам в логах при каждой попытке чтения/записи токенов.

## Решение
Добавлена проактивная проверка доступности Keychain при первом обращении к SecureLocalStorage.

### Изменения в `lib/data/local/secure_local_storage.dart`

1. **Новый метод `_checkKeychainAvailability()`**
   - Выполняет тестовую запись/чтение в Keychain
   - Если успешно — использует SecureStorage
   - Если ошибка — автоматически переключается на SharedPreferences
   - Проверка выполняется только один раз при первом обращении

2. **Обновлена логика миграции**
   - Перед миграцией токенов проверяется доступность Keychain
   - Если недоступен — токены остаются в SharedPreferences
   - Миграция помечается как завершённая, чтобы избежать повторных попыток

3. **Обновлены все методы работы с токенами**
   - `setToken()`, `getToken()`, `setRefreshToken()`, `getRefreshToken()`
   - Все методы проверяют доступность Keychain при первом вызове
   - После проверки используют соответствующее хранилище без повторных попыток

## Результат
- ✅ На iOS симуляторе: нет ошибок Keychain в логах, используется SharedPreferences
- ✅ На реальных iOS устройствах: используется Keychain (secure storage)
- ✅ На Android: используется EncryptedSharedPreferences (secure storage)
- ✅ Миграция токенов работает корректно на всех платформах
- ✅ Fallback механизм работает прозрачно для пользователя

## Тестирование
1. Запустить на iOS симуляторе — не должно быть ошибок Keychain
2. Запустить на реальном iOS устройстве — должен использоваться Keychain
3. Проверить миграцию токенов (удалить приложение, переустановить с токенами в SharedPreferences)
4. Проверить login/logout flow

## Технические детали
- Тестовый ключ: `_keychain_availability_test`
- Проверка выполняется асинхронно при первом обращении
- Результат кэшируется в `_keychainAvailabilityChecked` флаге
- Fallback флаг: `_useSharedPreferencesFallback`
